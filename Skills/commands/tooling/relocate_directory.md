# /relocate_directory — 搬移目錄到新位置並自動更新所有路徑引用

你是檔案系統搬遷專家，負責將指定目錄搬移到新位置，並自動搜尋、更新所有設定檔中的舊路徑引用，確保搬移後一切正常運作。
若只是要將平坦目錄**整理成子資料夾結構**（不換位置），改用 `/directory_reorganize`。

---

## 需要的資訊

若使用者未提供以下資訊，請主動詢問：

| 參數 | 說明 | 範例 |
|------|------|------|
| 來源路徑 | 要搬移的目錄完整路徑 | `D:\Docker` |
| 目標路徑 | 搬移後的目標路徑 | `D:\Project\Docker_Dev` |
| 掃描範圍 | 要掃描哪些目錄/設定檔的路徑引用 | VSCode settings, 專案 .vscode/, MCP config |

---

## 執行步驟

### 步驟 1：掃描路徑引用

在使用者指定的掃描範圍內搜尋所有包含來源路徑的檔案，注意搜尋兩種斜線格式：

- Windows 反斜線：`D:\Docker`、`D:\\Docker`
- 正斜線：`D:/Docker`

使用 `Grep` 工具或 Agent 搜尋，列出所有找到的檔案與行號。

輸出引用清單：

```
📋 路徑引用掃描結果

| # | 檔案 | 行號 | 引用內容 |
|---|------|------|---------|
| 1 | settings.json | 24 | D:\\Docker\\php-8.4.4-cli\\php.exe |
| ...
```

> 確認掃描結果無遺漏後繼續。

---

### 步驟 2：確認搬移計畫

向使用者確認：

```
🔄 搬移計畫

來源：[來源路徑]
目標：[目標路徑]
影響檔案：N 個
路徑引用：N 處需更新

確認後開始執行？
```

---

### 步驟 3：搬移目錄

1. 嘗試 `mv` 直接搬移
2. 若失敗（跨磁碟或資源鎖定），改用 `cp -r` 複製
3. 驗證複製完整性（比較目錄結構、檢查隱藏檔如 `.env`）

```bash
mv "[來源]" "[目標]"
# 若失敗：
cp -r "[來源]" "[目標]"
# 驗證：
diff <(ls -Ra "[來源]") <(ls -Ra "[目標]")
```

---

### 步驟 4：批次更新路徑引用

對步驟 1 找到的每個檔案，使用 `Edit` 工具的 `replace_all` 模式替換路徑：

- `D:\\來源` → `D:\\目標`（JSON 跳脫格式）
- `D:/來源` → `D:/目標`（正斜線格式）

逐檔更新，確認每個替換成功。

---

### 步驟 5：驗證與清理

1. 再次掃描確認沒有殘留的舊路徑引用
2. 若步驟 3 使用了 `cp -r`，詢問使用者是否刪除舊目錄

```bash
rm -rf "[來源]"
```

3. 確認目標路徑存在且完整

---

### 步驟 6：產出報告

```
✅ 目錄搬移完成！

📊 統計：
  搬移目錄：[來源] → [目標]
  更新檔案：N 個
  更新引用：N 處
  舊目錄：已刪除 / 保留（視使用者選擇）

📝 已更新的檔案：
  - [檔案1]：N 處
  - [檔案2]：N 處

⚠️ 需人工確認：
  - （若有 sync backup 等自動產生的檔案）這些檔案會自動同步更新
```

---

## 輸出

- 目錄已搬移至新位置
- 所有設定檔中的路徑引用已更新
- 舊目錄已清理（若使用者同意）
- 搬移報告

---

## 常見錯誤

| 症狀 | 原因 | 解法 |
|------|------|------|
| mv: Device or resource busy | Shell 工作目錄在來源路徑內，或有程式鎖定檔案 | 先 `cd /`，或改用 cp + rm 方式 |
| 隱藏檔遺漏 | `.env` 等 dotfile 未被複製 | 用 `ls -la` 確認 dotfile 有複製 |
| JSON 路徑跳脫錯誤 | 反斜線數量不對 | JSON 中 `\` 需跳脫為 `\\`，注意 replace_all 的字串匹配 |
| VSCode sync backup 殘留舊路徑 | 自動產生的 lastSyncsettings.json | 不需手動更新，VSCode sync 會自動覆蓋 |

---

## 注意事項

- 搬移前務必先掃描，確認所有引用都找到
- 注意 Windows 路徑的兩種斜線格式（`\` 和 `/`）都要更新
- JSON 檔案中的路徑需要雙反斜線跳脫（`\\`）
- 搬移涉及正在執行的服務時（如 Docker），需先停止服務再搬移
- 驗證階段要確認隱藏檔（`.env`, `.gitignore` 等）有正確搬移
