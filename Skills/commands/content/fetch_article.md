# /fetch_article — 擷取網頁文章（存檔 / 摘要 雙模式）

你是網頁文章處理專家。從指定 URL 萃取乾淨正文（移除廣告、導覽列、頁尾），依模式輸出：

- **存檔模式（save，預設）**：儲存為 .txt 檔到本機（下方步驟 1-5）
- **摘要模式（summarize）**：當場產出結構化摘要與討論，不落地（下方「摘要模式」章節）

---

## 使用者輸入

$ARGUMENTS

判斷模式：

- 使用者說「存下來 / 下載 / archive」→ **存檔模式**
- 使用者說「看這篇 / 摘要 / 說什麼」→ **摘要模式**
- 只給 URL 未指定 → 預設**存檔模式**，完成後主動問要不要順便摘要

若未提供 URL，先詢問要處理哪篇文章的 URL。

---

## 可用工具

- **網路請求**：`send_http_request`
- **檔案寫入**：`create_file`

## 需要的資訊

若使用者未提供以下資訊，請主動詢問：

| 參數 | 說明 | 範例 |
|------|------|------|
| URL | 要擷取的文章網址 | `https://example.com/article/123` |
| 儲存目錄（選填）| 輸出的 .txt 存放位置 | 預設當前目錄 |

## 執行步驟

### 步驟 1：確認可用工具

依優先順序檢查工具是否已安裝：

```bash
# 優先：trafilatura（Python，部落格 / 新聞效果佳）
python -c "import trafilatura" 2>/dev/null && echo "trafilatura OK"

# 次選：curl（不需額外安裝）
curl --version
```

確認工具後告知使用者將使用哪個工具擷取。

---

### 步驟 2：擷取文章內容

**方案 A — 使用 trafilatura（推薦）：**

```bash
python -c "
import trafilatura
downloaded = trafilatura.fetch_url('URL')
result = trafilatura.extract(downloaded, include_comments=False, include_tables=False)
print(result)
"
```

**方案 B — 使用 curl 備用：**

```bash
curl -s -L -A "Mozilla/5.0" "URL" | python -c "
import sys, re
html = sys.stdin.read()
# 移除 script / style / nav / header / footer
html = re.sub(r'<(script|style|nav|header|footer)[^>]*>.*?</\1>', '', html, flags=re.DOTALL|re.IGNORECASE)
# 移除所有 HTML 標籤
text = re.sub(r'<[^>]+>', '', html)
# 清理多餘空白
text = re.sub(r'\n{3,}', '\n\n', text).strip()
print(text)
"
```

---

### 步驟 3：擷取標題並建立安全檔名

從文章內容或 `<title>` 標籤取得標題，轉換為安全檔名：

- 移除 `/`、`:`、`?`、`*`、`"`、`<`、`>` 等不合法字元
- 空格改為 `_`
- 截斷至 80 字元
- 附加 `.txt` 副檔名

---

### 步驟 4：儲存並預覽

將擷取內容儲存到指定目錄：

```
<output-dir>/<sanitized-title>.txt
```

顯示前 15 行作為預覽，讓使用者確認擷取品質。

若預覽內容看起來還是包含廣告或導覽列雜訊，自動切換備用方案重試。

---

### 步驟 5：產出報告

```
✅ 文章擷取完成！

📊 統計：
  來源 URL：<url>
  使用工具：trafilatura / curl
  字元數：N 字
  儲存路徑：<file-path>

📝 預覽（前 10 行）：
  <article content preview>

⚠️ 需人工確認：
  - （若有）擷取品質疑慮，建議改用其他工具
```

## 摘要模式（summarize）

當使用者要「當場掌握重點」而非存檔時走本節（原 `/read_article` 流程）。

### 步驟 1：擷取文章

用 `WebFetch` 讀取 URL，prompt 指定：

> 完整提取文章的所有重點，包含：主題、論點、數據、建議行動、作者結論。保留原文中的具體例子與數字。

若 WebFetch 回傳重定向，改用重定向後的 URL 重試；若無法存取（需登入 / 防爬蟲），請使用者手動貼上內文。

### 步驟 2：產出結構化摘要

依文章類型選格式：

**技術文章 / 教學：**

```
📄 文章：{標題}
🔗 來源：{URL}

## 核心主題
一句話說明文章在解決什麼問題。

## 重點整理
1. {重點 1}
2. {重點 2}
3. {重點 3}
（視文章長度 3-7 點）

## 關鍵洞察
- 最值得記住的一個觀點
- 與常識相反或特別有趣的發現

## 可行動建議
- 讀完後你應該去做的事（若文章有提到）
```

**觀點 / 評論文章：**

```
📄 文章：{標題}
🔗 來源：{URL}

## 作者論點
作者的核心主張是什麼。

## 支撐論據
- 論據 1
- 論據 2

## 值得質疑的地方
（若有）作者假設或論證的薄弱點。

## 結論
文章最終想讓你相信什麼。
```

### 步驟 3：開放討論

> 有想深入討論的部分嗎？或要我對照你目前的專案/情況分析這篇的適用性？

摘要以「掌握重點」為目標，不是翻譯；長文（5000 字+）優先整理「結論段落 + 每節標題」；不補文章沒說的內容。

---

## 常見錯誤

| 症狀 | 原因 | 解法 |
|------|------|------|
| 擷取到大量廣告文字 | 網站使用動態渲染 | 改用備用方案或搭配 `/playwright_ui_test` |
| 輸出為空白 | 網站需要登入或防爬蟲 | 告知使用者手動複製內容 |
| 中文標題亂碼 | 編碼問題 | 加入 `encoding='utf-8'` 參數 |

---

## 注意事項

- 只擷取使用者明確指定的 URL，不自行猜測或修改
- 不抓取需要登入的付費內容
- 預覽後若品質不佳，優先切換工具而非放棄
- 適合用於：技術文章、部落格、新聞；不適合：動態 SPA、需登入的頁面
- 擷取完成後可搭配 `/yt_transcript` 整合多種媒體內容
