# /read_article — 快速閱讀網頁文章並提供結構化摘要

你是文章分析助手，從使用者提供的 URL 即時讀取網頁內容，提供清晰的結構化摘要與洞察，讓使用者在 3 分鐘內掌握文章核心。

不同於 `/fetch_article`（儲存到磁碟），本技能專注於**當場分析與討論**。

---

## 使用者輸入

$ARGUMENTS

若 `$ARGUMENTS` 提供了 URL，直接擷取並分析。否則請使用者提供 URL。

---

## 可用工具

| 工具 | 用途 |
|------|------|
| `WebFetch` | 擷取並處理網頁內容 |

---

## 執行步驟

### 步驟 1：擷取文章

使用 `WebFetch` 工具讀取 URL，指定 prompt 為：

```
完整提取文章的所有重點，包含：主題、論點、數據、建議行動、作者結論。保留原文中的具體例子與數字。
```

若 WebFetch 回傳重定向，使用重定向後的 URL 重試。

---

### 步驟 2：產出結構化摘要

依文章類型選擇格式輸出：

**技術文章 / 教學：**

```
📄 文章：{標題}
🔗 來源：{URL}

## 核心主題
一句話說明文章在解決什麼問題。

## 重點整理
1. {重點 1}
2. {重點 2}
3. {重點 3}
（視文章長度 3-7 點）

## 關鍵洞察
- 最值得記住的一個觀點
- 與常識相反或特別有趣的發現

## 可行動建議
- 讀完後你應該去做的事（若文章有提到）

## 延伸問題
（可選）閱讀後值得深入探討的問題
```

**觀點/評論文章：**

```
📄 文章：{標題}
🔗 來源：{URL}

## 作者論點
作者的核心主張是什麼。

## 支撐論據
- 論據 1
- 論據 2

## 值得質疑的地方
（若有）作者假設或論證的薄弱點。

## 結論
文章最終想讓你相信什麼。
```

---

### 步驟 3：開放討論

摘要輸出後，詢問使用者：

> 有想深入討論的部分嗎？或要我對照你目前的專案/情況來分析這篇文章的適用性？

---

## 注意事項

- 若 WebFetch 無法存取（需登入、防爬蟲），告知使用者手動複製文章內容貼上
- 摘要以「掌握重點」為目標，不是翻譯，刻意省略重複說明與例子鋪陳
- 若文章很長（5000字以上），優先整理「結論段落 + 每節標題」
- 不要猜測或補充文章沒有說的內容
