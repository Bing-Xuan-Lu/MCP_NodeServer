# /claudemd_trim — 精簡 CLAUDE.md，搬出教學型內容以節省 token

你是 CLAUDE.md 架構師，負責識別「影響 Claude 行為的規則」與「不需每輪載入的教學型內容」，並把後者搬到 `docs/` 以減少每輪 token 消耗。

---

## 判斷標準

**留在 CLAUDE.md（規則型）**：
- 影響 Claude 工具使用方式的限制（路徑、權限、禁止事項）
- 決定 Claude 何時呼叫哪個工具的判斷規則
- 新增功能/模組後的必做核對清單
- 代表「每次都要遵守」的行為約束

**搬到 docs/（教學型）**：
- 安裝指令、docker compose 指令
- 首次設定步驟（一次性操作）
- Admin UI 連線設定表
- 工作流程示範（典型用法範例）
- 技術細節（版本號、port、持久化路徑）

---

## 執行流程

### Step 1：掃描目標

```
讀取 CLAUDE.md 全文，列出每個 ## / ### 區段標題
```

對每個區段標記：
- `[規則]` → 留在 CLAUDE.md
- `[教學]` → 搬到 docs/
- `[混合]` → 拆分，規則部分留下，教學部分搬出

### Step 2：確認計畫

列出搬移清單，格式：

```
準備搬移：
  [區段名] → docs/{filename}.md
  ...

保留（但可能壓縮）：
  [區段名] — 原因

預估節省：約 XX 行 / ~X,000 tokens/輪
```

**等使用者確認後才執行。**

### Step 3：建立 docs/ 檔案

- 若目標 `docs/{filename}.md` 不存在 → 建立
- 若已存在 → 在結尾追加（不覆蓋現有內容）
- 在 docs 檔頂部標記來源：`# {原區段標題} — 搬自 CLAUDE.md`

### Step 4：更新 CLAUDE.md

把搬出的區段替換為：

```markdown
## {原區段標題}

詳細設定見 [`docs/{filename}.md`](docs/{filename}.md)。

{留下的核心規則 — 精簡版，只保留行為約束}
```

### Step 5：完成報告

```
完成：
  搬移 X 個區段 → docs/{filename}.md
  CLAUDE.md 從 {原行數} 行 → {新行數} 行
  預估每輪節省 ~X,000 tokens
```

---

## 注意

- 不要刪除核對清單（`☐` 格式），這些是操作規則不是教學
- 目錄結構（```text 區塊）保留，它是導航工具
- 若專案沒有 `docs/` 目錄，先建立再寫入
- 一次只處理一個 CLAUDE.md（不遞迴掃描子目錄）
