# frontend_qc — PM 迭代循環

> 此檔案由 `/frontend_qc` 步驟 5 讀取，報告產出後自動進入此循環。

**報告產出後立即進入 PM 循環，不需使用者手動重跑。**

---

```
=== PM 迭代循環 ===

[每輪結束後執行]

1. 統計當前報告的未關閉 Bug 數（P0/P1/P2/P3 合計）
   → 若全部 = 0 → 輸出最終驗收通過通知（結束循環）
   → 若 > 0 → 繼續步驟 2

2. 產出「工程師修正任務單」（格式見下方）

3. 輸出任務單後詢問工程師：
   「以上 N 項 Bug 已整理。請修正後回覆「完成」或指定複驗範圍（如：完成 1.1, 1.3, 3.2）。」

4. 工程師回覆後，僅對「已修正項目」重跑對應維度：
   - VISUAL Bug → 重新執行 2a（強制重讀設計稿，不看上一輪截圖）
   - FIELD Bug  → 重新執行 2b（強制重讀規格書，不憑記憶）
   - FUNC Bug   → 重新執行 2c（重新操作互動）
   - DATA Bug   → 重新執行 2d
   - RWD Bug    → 重新執行 2e（重新 resize 截圖）

5. 對比前次結果：
   - 通過 → 標記 ✅ 已修正，從 Bug 清單移除
   - 仍有問題 → 標記 ❌ 修正未通過（說明仍存在的差異）
   - 未修正 → 保留原狀

6. 輸出版次遞增的新報告（v1 → v2 → v3...）
   → 回到步驟 1
```
