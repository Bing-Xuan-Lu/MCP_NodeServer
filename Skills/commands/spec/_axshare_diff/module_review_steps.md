# 模組校閱模式 — 步驟 B1-B6

> 此檔案由 `/axshare_diff` 在模組校閱模式時讀取。

適用場景：複雜模組（如訂單、會員）有多個子目錄和規格頁面，需要系統性逐頁比對。

---

### 步驟 B1：建立頁面清單

```
1. 掃描模組目錄結構：
   ls adminControl/{module}/          → 主目錄檔案
   ls adminControl/{module}/*/        → 所有子目錄

2. 讀取本地規格書索引，篩選該模組的所有頁面：
   Read spec/axshare_spec_reference_backend.md
   → Grep 模組關鍵字（如「訂單」「order」）
   → 列出所有對應的規格書頁面 ID + 名稱

3. 建立「頁面對照表」：

   | # | 規格書頁面 | page ID | 對應程式檔案 | 子目錄 | 來源 |
   |---|-----------|---------|-------------|--------|------|
   | 1 | 全部/列表 | jj46fr | order/list.php | order/ | 主要 |
   | 2 | 待處理/列表 | abc123 | order/pending/list.php | order/pending/ | 主要 |
   | 3 | 出貨狀態說明 | def456 | — | — | 📎 跨頁引用 |

4. **跨頁引用展開**：掃描步驟 2 篩出的所有頁面，檢查「**跨頁引用**」區塊（或全文搜尋「參照」「詳見」「見」「連結」「同 XX 頁面」）。將被引用的頁面加入對照表，來源標記為「📎 跨頁引用」。只展開一層。

5. 向使用者確認頁面清單是否完整，是否需要增減（含跨頁引用的頁面）
```

---

### 步驟 B2：並行擷取規格書頁面

```
1. 將規格書頁面分組（最多 4 組 Agent 並行）
   - 分組規則同步驟 A2

2. 每個 Agent 任務：
   - 逐頁 browser_navigate(AxShare URL #id=xxx&p=yyy)
   - browser_snapshot() → 擷取 accessibility tree
   - 從 snapshot 提取：
     a. 搜尋區欄位（下拉選單選項、輸入框、按鈕）
     b. 列表欄位（table th 標頭、欄寬）
     c. 功能按鈕（匯出、封存、批次操作等）
     d. 特殊邏輯（彈窗、條件顯示、狀態轉換）
   - browser_take_screenshot(fullPage: true)
     → 存至 axshare_screen_shot/spec_{module}_{page_name}.png

3. 回傳結構化資料（每頁一份）：
   頁面名稱 | 搜尋區欄位[] | 列表欄位[] | 功能按鈕[] | 特殊邏輯[]
```

> 本地索引有文字摘要但**不含 layout 結構**，搜尋區/列表欄位仍需 Playwright 截圖確認。

---

### 步驟 B3：擷取測試網站頁面

```
1. 登入測試網站（若未登入）
2. 逐頁擷取（可並行，但注意 session）：
   - browser_navigate(測試網站 URL)
   - browser_snapshot() → 取得實際渲染的 accessibility tree
   - 提取同 B2 的四個維度

3. 額外檢查 DB 狀態：
   - 相關資料表是否有資料（空表 = 功能不可見）
   - 缺少的欄位（影響搜尋/顯示）
   - docker exec dev-mariadb mysql -uroot --skip-password -e "USE dbox3; SELECT COUNT(*) FROM {table};"
```

---

### 步驟 B4：逐頁比對

對每個頁面，比對四個維度：

```
維度 1 — 搜尋區：
  規格書有哪些搜尋欄位？測試網站有哪些？
  下拉選單的選項是否一致？

維度 2 — 列表欄位：
  表頭(th)數量和名稱是否一致？
  欄位順序是否相同？
  是否有互動功能差異（如「訂單編號」應可點擊但目前是純文字）？

維度 3 — 功能按鈕：
  匯出/封存/批次操作等功能是否存在？
  是否因 DB 空表而隱藏？

維度 4 — 特殊邏輯：
  狀態轉換、彈窗確認、條件顯示等
```

結果標記：✅ 一致 / ⚠️ 差異 / ❌ 缺少 / ➕ 多出

---

### 步驟 B5：產出模組校閱報告

讀取 `_axshare_diff/module_report_template.md` 作為報告骨架，填入比對數據後輸出至 `spec/{module}_diff_report.md`。

報告包含：總覽表（✅⚠️❌統計）→ 逐頁四維度比對 → P1/P2/P3 優先級矩陣 → DB 狀態檢查 → 統計。

> 📎 跨頁引用的頁面在報告中獨立成「跨頁引用規格」區段，與主要頁面分開統計，避免混淆主模組的完成度數字。

---

### 步驟 B6：向使用者報告

```
展示報告摘要：
- 總頁面數 / 比對項目數
- P1 數量（最優先處理）
- DB 狀態異常（空表等）
- 報告檔案路徑

詢問使用者是否要：
1. 開始修正 P1 項目
2. 調整優先級
3. 進一步深入某個子頁面
```
