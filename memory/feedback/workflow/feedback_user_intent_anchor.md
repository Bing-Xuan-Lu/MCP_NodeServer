description: 使用者連續糾正方向時停下實作，改為先把發現寫入規格/backlog，避免半途實作淪為廢碼
type: feedback
---

使用者在同一項任務上連續三次以上「糾正方向」（不是細節修正）時，停下手上實作。從「寫代碼 / 改資料」切換為「寫規格 / 附 backlog」模式，等方向確定再一次實作。

**Why**：

- 2026-05-14 PG_dbox3 autocalc 對齊：「重生 baseline → 不該重生 → 改 H9/K9 → 改 B5/G5 → 改全頁審計」，每輪連續轉向，中間所有實作幾乎全部籣為廢碼。
- 使用者連續轉向代表他還在探索問題軟其本身，不是推進某個明確解法。這時忝生產出實作是「為實作而實作」。

**How to apply**：

1. 偵測條件：同一任務 30 分鐘內使用者說「不是這個」「改成 X」「刪掉」「不對，改為 Y」 3 次以上。
2. 停下任何 code edit / SQL / Sheet 寫入，主動告知：「方向變動太多，先把目前所有發現寫進 [[improvements_backlog]] 或規格文件，等你確定方向再動手。」
3. 這段期間只做唯讀作業：Grep / Read / 讀 schema / `gsheet_fetch_with_state` 唯讀 fetch。不動資源。
4. 使用者明確「好，這個方向」後才重起實作。
5. 本規則只適用「方向轉向」，不適用于「誤打字」「介面調整」等細節糾正（那些直接改即可）。
