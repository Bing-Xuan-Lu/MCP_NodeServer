🏠 助理員工的辦公環境準備 (Setup SOP)

  • Git & Node.js 18+ 安裝並加入系統 PATH
  • Docker Desktop 必須安裝並啟動
  • 書籤工具預設路徑：C:\Users\YourName\AppData\Local\Google\Chrome\User Data\Default
  • Windows 執行 .\setup.ps1，若被攔截請先執行 Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
  • 工作目錄預設 D:\Project，若無 D 槽請修改 config.js
  • 建議安裝 MCP Inspector 擴充功能除錯

🐳 Docker 選用元件

以下元件皆為選用，不啟用時其他工具完全不受影響。需先確認 Docker Desktop 執行中。

🐍 Python 執行環境
提供 run_python_script 工具，在 Docker 容器中執行 Python 腳本。
啟動（一次性）：
cd python && docker compose up -d
容器名稱：python_runner（Python 3.12-slim）
設為 restart: unless-stopped，Docker 執行中即自動常駐。
安裝額外套件：docker exec python_runner pip install 套件名稱
🔍 RAG 向量檢索（ChromaDB）
提供 rag_index / rag_query / rag_status 工具，從大量檔案中語意搜尋最相關的程式碼片段。
啟動（一次性）：
cd chromadb && docker compose up -d
容器名稱：dev-chromadb（port 8010）
設為 restart: unless-stopped，Docker 執行中即自動常駐。
使用內建 Embedding 模型（all-MiniLM-L6-v2，免費、本地執行）。
Admin UI：http://localhost:3100/
首次開啟進入 Setup 頁，Connection string 填 http://chromadb:8000，其餘預設，按 Connect。

🔍 RAG 使用流程

啟動 ChromaDB 後，只需兩步即可開始使用語意搜尋。

① 為專案建立索引
在 Claude 對話中告訴助理「索引 {ProjectFolder} 專案」，或直接呼叫：
rag_index { project: "{ProjectFolder}", paths: ["{ProjectFolder}/"] }
首次完整掃描，之後自動增量（僅處理變更檔案）。加 force: true 強制全部重建。
② 之後完全自動
你只需用自然語言提問，Claude 會自動判斷是否呼叫 rag_query 搜尋相關程式碼。
rag_query { project: "{ProjectFolder}", query: "訂單折扣邏輯" }
查看索引狀態：rag_status {} 列出所有 collections。
Collection 策略
每個專案獨立 collection（rag_{ProjectFolder}），另有共用 rag_shared 供跨專案知識。
索引到共用：加 shared: true
查詢含共用：加 include_shared: true

部署流程 · Deployment Pipeline

從本機開發完成到推上測試機的標準流程，支援直連 DB 與間接（SFTP+PHP）兩種 DB 存取方式。

環境架構
本機開發環境
D:\Project\{ProjectFolder}
Git 版控 · 程式碼為主
→
內部測試機
/var/www/html_{project}
SFTP + DB 直連
→
準測試機
/var/www/html_{project}
SFTP + DB 間接（PHP 腳本）
  • Phase A — 安全閘
    REMOTE DIFF

    比對本機 vs 遠端檔案，找出同事直接 SFTP 修改過的檔案，避免覆蓋。

    1. SFTP 拉取遠端檔案 → diff /remote_diff
    2. 有衝突 → 決定覆蓋或 merge
  • Phase B — 程式碼部署
    SFTP UPLOAD

    以本機為主上傳程式碼，自動排除設定檔與非程式資源。

    1. SFTP 上傳（排除 config/.env/uploads） /sftp_deploy
  • Phase C — DB Migration
    DATABASE UPDATE

    依 DB 存取方式自動分流：直連用 execute_sql，無法直連用 SFTP+PHP 間接執行。

    直連（內部測試機） /db_migration
    間接（準測試機） /remote_db_exec
  • Phase D — 驗證
    SMOKE TEST + REGRESSION

    確認部署後網站正常，並走一遍核心業務路徑確保主幹未斷。

    1. send_http_request 首頁 → 200
    2. 黃金路徑回歸驗證 /e2e_golden_path
    3. 失敗時查 error log /sftp_ops
一鍵整合
以上四個 Phase 串成一條指令
/full_deploy
支援 code-only、db-only、--skip-diff 參數

MCP Server 自檢流程 · MCP Health Check Pipeline

每次重啟或更新 MCP Server 後執行，確保工具完整、路徑可達、Skills 已同步、Dashboard 一致。

  • ① 伺服器啟動
    SERVER STARTUP

    確認 index.js 正常啟動，無 module not found 錯誤。

    node index.js → 無錯誤輸出
    Claude Code 重啟後 MCP 連線恢復
  • ② 工具完整性
    TOOLS INTEGRITY

    確認所有 tools/*.js 模組均已載入，工具數量與 CLAUDE.md 一致。

    12 個模組全部無報錯
    工具清單核對 CLAUDE.md tools/ 目錄
  • ③ 路徑與連線
    PATH & CONNECTIONS

    確認 basePath 可存取，按需開放額外路徑並驗證 DB 連線。

    basePath D:\Project\ 可讀寫
    DB：load_db_connection → set_database 測試
    額外路徑：grant_path_access（需時再開）
  • ④ Skills 同步
    SKILLS SYNC

    確認 ~/.claude/commands/ 與 Skills/commands/ 同步，數量未超出 60 個上限。

    1. 審查超載或重疊 /skill_audit
    2. SKILL_INDEX.md 與 commands/ 比對一致
  • ⑤ Dashboard 核對
    DASHBOARD AUDIT

    確認 tag 數、dept-count、section-total、JS SKILLS 物件均與實際 Skill 數量吻合。

    1. 工具設置評分報告 /harness_audit
    2. dashboard.html tag/count/JS 人工核對清單
