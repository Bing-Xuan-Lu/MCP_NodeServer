# Skill 評估測試案例（Eval Cases）

評估框架：驗證 Skill 在特定輸入下是否產生預期行為。
用途：Skill 改動後快速 Regression 測試。

---

## 格式說明

| 欄位 | 說明 |
|------|------|
| Prompt | 實際輸入給 Claude 的指令 |
| 預期行為 | Claude 應採取的動作（關鍵步驟） |
| 驗證點 | Pass/Fail 判斷依據 |

---

## PHP 開發部

### /php_crud_generator

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/php_crud_generator orders` | 詢問 DB 連線，執行 get_db_schema | 有呼叫 set_database 或詢問 DB 資訊 |
| 2 | `/php_crud_generator` (無參數) | 詢問資料表名稱 | 必須問使用者提供更多資訊 |
| 3 | 提供資料表後 | 列出即將產生的檔案清單，等確認 | 不應直接開始寫檔，先等確認 |

### /php_upgrade

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/php_upgrade` | 詢問專案路徑或自動掃描 | 使用 list_files 掃描 .php 檔案 |
| 2 | 掃描結果後 | 列出所有需修正的語法問題 | 應分類：Deprecated 函式、型別問題等 |

## 測試品管部

### /php_crud_test

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/php_crud_test module_a` | 詢問 DB 連線與專案路徑 | 必須在測試前確認連線 |
| 2 | 提供資訊後 | 先讀取 Model class 再動手 | 不可盲測，需先分析程式碼 |
| 3 | 新增測試後 | 呼叫 execute_sql 驗證 DB | 每次寫入後必須查詢 DB 確認 |
| 4 | 測試完成後 | 清理測試資料並產出報告 | 確認沒有遺留 MCP_TEST_ 前綴資料 |

### /playwright_ui_test

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/playwright_ui_test` (無參數) | 詢問模式（測試/除錯）、容器、網址 | 必須問到 6 個必要參數 |
| 2 | 選「測試模式」 | 讀取步驟參考檔後開始登入 | 應主動讀取 playwright_ui_test_steps.md |
| 3 | 登入成功後 | 截圖並繼續下一步 | 截圖名稱格式正確（01_login_success.png）|
| 4 | 選「除錯模式」 | 確認 Xdebug 環境後才開始操作 | 必須提醒 VSCode Listen + 設斷點 |

## 開發流程部

### /git_commit

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/git_commit` | 執行 git status + git diff | 應分析 staged 和 unstaged 差異 |
| 2 | 有 staged 變更時 | 產生繁體中文 commit 訊息 | 訊息應為繁體中文，有動詞開頭 |
| 3 | 有 unstaged 變更時 | 詢問是否加入或跳過 | 不應自動 git add，需確認 |
| 4 | 訊息產生後 | 詢問確認後才執行 commit | 不應自動 commit，需使用者確認 |

### /clean_arch

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/clean_arch` | 詢問要審查哪個模組/目錄 | 不應直接掃描整個專案 |
| 2 | 提供模組後 | 讀取相關檔案後提出架構問題 | 應識別 God Class、業務邏輯位置等 |
| 3 | 提出問題後 | 給出具體重構建議（不直接改碼）| 建議應有優先順序 |

### /sadd

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/sadd spec.md` | 讀取規格書後列出任務清單 | 任務應區分「循序」和「可並行」 |
| 2 | 確認任務後 | 逐一派遣 SubAgent，每次完成審查後才繼續 | 不應一次派遣所有任務 |

### /learn_claude_skill

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|

## 內容擷取部

### /read_article

| # | Prompt | 預期行為 | 驗證點 |
|---|--------|---------|--------|
| 1 | `/read_article https://example.com/article` | 呼叫 WebFetch 擷取內容 | 應直接擷取，不問多餘問題 |
| 2 | 擷取成功後 | 輸出結構化摘要 | 必須有「核心主題」和「重點整理」區塊 |
| 3 | 擷取失敗（需登入）| 告知限制並建議手動貼上 | 不應靜默失敗或輸出錯誤內容 |
| 4 | 摘要後 | 開放討論，詢問是否要深入某個點 | 摘要完後不應就此結束 |

---

## 維護說明

- 新增 Skill 時：同步在此加入 3-5 個基本測試案例
- 修改 Skill 時：先執行此清單確認無 regression
- 每季審查：移除已棄用技能的測試案例，新增新案例
